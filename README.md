## 🔍 Overview

The **Real-Time Stress Monitoring System** combines computer vision, natural language processing, and generative AI to provide a comprehensive mental wellness platform. Users can start monitoring sessions that capture facial expressions via webcam, analyze typed text for emotional cues, and receive AI-generated wellness recommendations tailored to their current stress state.

The system features a rich interactive dashboard with real-time graphs, emotion breakdowns, stress-level indicators, a location-based consultant finder, and an AI chatbot powered by Google Gemini for ongoing mental health support.

---

## ✨ Key Features

| Feature | Description |
|---------|-------------|
| **🎥 Facial Emotion Recognition** | Real-time webcam-based emotion detection using MediaPipe Face Mesh and a trained ML classifier (7 emotions: happy, sad, angry, fear, surprise, disgust, neutral) |
| **📝 Text Stress Analysis** | NLP-based sentiment and stress analysis on user-typed text using Hugging Face Transformers with Gemini AI augmentation |
| **⌨️ Keystroke Dynamics** | Monitors typing patterns (speed, pauses, pressure) as behavioral indicators of stress |
| **📊 Live Dashboard** | Interactive dashboard with real-time emotion graphs, stress score gauges, and session history powered by Recharts |
| **🤖 AI Wellness Recommendations** | Personalized, context-aware wellness suggestions generated by Google Gemini 2.0 Flash based on detected stress levels and emotions |
| **📍 Location-Based Consultant Finder** | Geolocation-powered search to find nearby mental health professionals and wellness centers with distance calculations |
| **💬 AI Chatbot** | Integrated conversational AI assistant (Gemini-powered) for real-time mental health support and guidance |
| **🔐 Authentication** | Secure user authentication and session management via Firebase Auth |
| **🎨 Modern UI/UX** | Sleek, animated interface built with Framer Motion, Tailwind CSS, and Lottie animations |

## 🚀 Getting Started

### Prerequisites

- **Python** 3.9+ ([download](https://www.python.org/downloads/))
- **Node.js** 18+ ([download](https://nodejs.org/))
- **Google Gemini API Key** ([get one](https://aistudio.google.com/apikey))
- **Firebase Project** with Authentication and Firestore enabled
- **Webcam** (for facial emotion recognition)

### 1. Clone the Repository

```bash
git clone https://github.com/sathvik-dvdg/Real-time-StressMonitoring-System.git
cd Real-time-StressMonitoring-System
```

### 2. Backend Setup

```bash
# Navigate to backend
cd backend

# Create and activate a virtual environment (recommended)
python -m venv venv
# Windows
venv\Scripts\activate
# macOS/Linux
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Configure environment variables
# Create a .env file in the backend/ directory:
echo GEMINI_API_KEY=your_gemini_api_key_here > .env
echo GEMINI_MODEL=gemini-2.0-flash >> .env

# Start the backend server
python app.py
```

The backend server will start at **http://localhost:5000**.

### 3. Frontend Setup

```bash
# Navigate to frontend (from project root)
cd frontend

# Install dependencies
npm install

# Start the development server
npm run dev
```

The frontend will be available at **http://localhost:5173**.

### 4. Open in Browser

Navigate to `http://localhost:5173` — sign in, start a monitoring session, and explore the dashboard!

## 📡 API Reference

All endpoints are prefixed with `/api` and accept/return JSON.

### Core Endpoints

| Method | Endpoint | Description | Rate Limit |
|--------|----------|-------------|------------|
| `POST` | `/api/process_face` | Analyze a webcam frame for facial emotions | Exempt |
| `POST` | `/api/process_text` | Analyze text for stress and emotional content | 60/min |
| `POST` | `/api/chat` | Send a message to the Gemini AI chatbot | 60/min |
| `GET`  | `/api/health` | Health check (server + Gemini status) | — |

### Wellness Endpoints

| Method | Endpoint | Description | Rate Limit |
|--------|----------|-------------|------------|
| `POST` | `/api/recommendations` | Get AI-generated wellness recommendations | 20/min |
| `POST` | `/api/recommendations/feedback` | Submit feedback on a recommendation | 30/min |
| `POST` | `/api/consultants/nearby` | Find nearby mental health professionals | 10/min |

### Example: Process Face

```bash
curl -X POST http://localhost:5000/api/process_face \
  -H "Content-Type: application/json" \
  -d '{"image": "<base64-encoded-image>"}'
```

**Response:**
```json
{
  "status": "success",
  "stress_score": 42,
  "emotion": "neutral",
  "detected_emotions": {
    "happy": 0.12,
    "sad": 0.08,
    "neutral": 0.65,
    "angry": 0.05,
    "fear": 0.03,
    "surprise": 0.04,
    "disgust": 0.03
  }
}
```

### Example: Get Recommendations

```bash
curl -X POST http://localhost:5000/api/recommendations \
  -H "Content-Type: application/json" \
  -d '{"stress_score": 75, "emotion_data": {"primary": "angry", "secondary": "sad"}}'
```

## 🔐 Environment Variables

### Backend (`backend/.env`)

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `GEMINI_API_KEY` | ✅ Yes | — | Google Gemini API key for AI features |
| `GEMINI_MODEL` | No | `gemini-2.0-flash` | Gemini model variant to use |

### Frontend (`frontend/.firebaserc`)

Firebase configuration is managed through the Firebase SDK setup in the `src/database/` directory. Ensure your Firebase project has:
- **Authentication** enabled (Google & Email providers)
- **Cloud Firestore** database created

## 🚢 Deployment

### Backend (Docker)

```bash
cd backend
docker build -t stress-monitor-backend .
docker run -p 5000:5000 --env-file .env stress-monitor-backend
```

### Frontend (Firebase Hosting)

```bash
cd frontend
npm run build
firebase deploy --only hosting
```

### Using Startup Scripts

**Windows:**
```bash
cd backend
start_server.bat
```

**Linux/macOS:**
```bash
cd backend
chmod +x start_server.sh
./start_server.sh
```
